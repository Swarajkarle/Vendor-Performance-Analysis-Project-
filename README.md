# Vendor Performance Analysis with Python & Power BI

## 📌 Project Overview
This project analyzes vendor performance using sales data to identify trends, profitability patterns, and operational insights. The workflow includes data ingestion, data cleaning, exploratory data analysis (EDA), automated reporting, and dashboard visualization using Power BI.

The final output of the project is a detailed report containing insights and recommendations to improve vendor efficiency and business performance.

## 📂 Repository Structure
├── data/
│   └── vendor_sales_summary.csv
├── notebooks/
│   ├── Exploratory_Data_Analysis.ipynb
│   └── Vendor_Performance_Analysis.ipynb
├── scripts/
│   ├── get_vendor_summary.py
│   └── ingestion_db.py
├── dashboard/

├── reports/

└── README.md

## 🛠️ Technologies Used
- Programming Language: Python  
- Libraries: Pandas, Matplotlib, Seaborn  
- Visualization Tool: Power BI  
- Development Environment: Jupyter Notebook  
- Data Format: Excel / CSV  
- Reporting: PDF Documentation  

## 🔑 Project Highlights
- Automated data ingestion and preprocessing for vendor sales data
- Detailed Exploratory Data Analysis (EDA) to identify patterns and trends
- Python scripts for generating automated vendor performance summaries
- Interactive and visually rich Power BI dashboard
- Professional analytical report summarizing key findings and insights

## ▶️ How to Run the Project
1. Clone the repository.
2. Open the Jupyter notebooks in the `/notebooks` folder.
3. Run the notebooks to perform data analysis and generate insights.
4. Execute `get_vendor_summary.py` to generate the vendor summary dataset.
5. Open the `.pbix` file in Power BI to explore the dashboard.
6. Review the final report in the `/reports` folder.
